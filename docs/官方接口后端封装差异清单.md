# 官方接口后端封装差异清单

## 一、目的

这份文档用于对照：

- 官方 `codex app-server` README 中列出的能力
- 当前项目 `apps/server` 已经封装为后端业务接口的能力

目标是明确：

1. 哪些官方能力已经进入本项目后端
2. 哪些能力还没有进入后端封装层
3. 后续后端接口补齐时的优先级顺序

## 四、当前后端尚未封装的官方接口

下面这些能力在官方 README 中存在，但当前项目后端尚未形成统一业务接口。

## 4.1 高价值能力，建议优先封装

- `thread/metadata/update`
- `thread/compact/start`
- `thread/rollback`
- `thread/unsubscribe`
- `thread/shellCommand`
- `thread/backgroundTerminals/clean`
- `thread/inject_items`

### 内存能力

- `thread/memoryMode/set`
- `memory/reset`

### 工具交互能力

- `tool/requestUserInput`

### 能力目录补齐

- `app/list`
- `plugin/read`
- `config/mcpServer/reload`

## 4.2 中优先级能力

### 独立命令执行

- `command/exec`
- `command/exec/write`
- `command/exec/resize`
- `command/exec/terminate`

说明：

- 这组接口适合后续做“独立终端面板”或“显式命令执行器”
- 当前项目很多命令相关能力仍然依赖线程内 turn 流程，而不是独立命令会话

### 文件系统能力

- `fs/readFile`
- `fs/writeFile`
- `fs/createDirectory`
- `fs/getMetadata`
- `fs/readDirectory`
- `fs/remove`
- `fs/copy`
- `fs/watch`
- `fs/unwatch`

说明：

- 这些接口后续对桌面版很有价值
- 适合做文件浏览、文件预览、受控编辑、文件监听等能力

## 六、建议优先级

建议后端封装按下面顺序推进。

## P1：最优先

- `thread/compact/start`
- `thread/rollback`
- `thread/backgroundTerminals/clean`
- `thread/unsubscribe`
- `tool/requestUserInput`
- `app/list`
- `plugin/read`
- `config/mcpServer/reload`

原因：

- 对当前线程产品形态直接有价值
- 和现有线程、审批、MCP、插件体系天然衔接
- 对桌面版与前端交互都有明显收益

## P2：第二优先级

- `command/exec*`
- `fs/*`
- `thread/shellCommand`
- `thread/metadata/update`
- `thread/inject_items`
- `thread/memoryMode/set`
- `memory/reset`

原因：

- 能显著扩展产品能力边界
- 但需要更清晰的前端交互设计与安全边界

## P3：第三优先级

- `thread/realtime/*`
- `plugin/install`
- `plugin/uninstall`
- `marketplace/add`
- `mcpServer/oauth/login`
- `windowsSandbox/setupStart`
- `feedback/upload`
- `config/read`
- `config/value/write`
- `config/batchWrite`
- `configRequirements/read`
- `experimentalFeature/enablement/set`
- `externalAgentConfig/detect`
- `externalAgentConfig/import`

原因：

- 更偏平台化能力、运维能力或长期增强方向
- 不属于当前主链路第一优先级

## 七、推荐落地顺序

建议后续按下面顺序推进：

1. 补线程高价值能力
2. 补工具交互与插件/MCP 目录能力
3. 补命令执行与文件系统能力
4. 再做实时会话与平台配置能力

## 八、与前端开发的关系

这份清单对应的是“官方接口还没进入后端”的缺口。

后续整体节奏建议是：

1. 先按这份文档补齐后端封装
2. 再对照 `前后端接口接入清单.md` 补前端
3. 最后进入 Electron 联调与打包阶段
